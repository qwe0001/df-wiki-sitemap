# Delta Force Wiki - サイトマップ自動生成スクリプト

Swiki（df.swiki.jp）のページ一覧を自動取得し、
`sitemap.xml` を生成・GitHubへ反映するGoogle Apps Script（GAS）です。

---

## このシステムの概要

1. GASのURL Fetchで、Swikiのページ一覧（ https://df.swiki.jp/?cmd=list ）をHTML形式で自動取得
2. URLと最終更新日を抽出
3. （このGithub上にある）既存の`sitemap.xml`と自動でマージ
4. GitHubに自動でPush（このリポジトリのmainブランチにpushします）
5. 差分を人間（Wiki管理人である筆者、もしくは後任の管理人）が、 **目視で確認**（Commit履歴から`sitemap.xml`のdiffを確認）
6. 【任意】GitHubに上がっている最新の`sitemap.xml`を手動でダウンロードして、必要に応じて内容を調整。（順番を並び替えたり、ページ優先度を調整したり）
7. `sitemap.xml`を手動更新した場合は、Githubのmainブランチに直接アップロードすること
8. swikiのサーバー管理者に、直接メールで送付する（「swiki 問い合わせ」でググれば、連絡先が出ます）
9. swikiのサーバー管理者が、後日、FTPでアップロードしてくれる。以上で作業完了。

Wiki管理人は、swikiサーバーに直接FTPアップロードする権限がなく、
サイトマップ自動生成プラグインも存在せず、導入権限もないため、
このような回りくどいやり方になっています。

5～9の工程は手動です。
特に、5の **目視確認は必須です。**
絶対に省略してはなりません。
AIに頼らず、必ず、あなたが、してください。
（サイトの品質管理は、Wiki管理人としての責務です。）

## 通常の運用手順（定期メンテナンス）

もし、後任のサイト管理人の方が、
ソースコードを読む自信がない場合は、
この手順だけ守って頂ければ、大丈夫です。

Wikiにページが追加・更新されたタイミングで実施します。
月1回程度が目安です。

### 1. GASスクリプトを開く

[Google Apps Script](https://script.google.com) にアクセスし、このプロジェクト（`df-wiki-sitemap`）を開く。

### 2. `updateSitemap` を実行する

エディタ上部の関数選択ドロップダウンで `updateSitemap` を選び、「実行」ボタンを押す。

完了すると、ログに以下のように表示される：

```
=== サイトマップ更新開始 ===
[1/5] Swikiページ一覧を取得中...
  → 234 件のURLを抽出
[2/5] GitHubから既存sitemap.xmlを取得中...
  → 既存エントリ数: 220 件
[3/5] バリデーション実行中...
  → バリデーション通過
[4/5] マージ処理中...
  → lastmod更新: 12 件、新規追加: 3 件
[5/5] GitHubにPush中...
=== 完了 ===
GitHubのCommit Diffで差分を確認してください。
```

### 3. GitHub上で差分を確認する

このリポジトリのトップページ → 最新のcommitをクリック → `sitemap.xml` の差分を確認する。

意図しない変更（大量削除など）がないかチェックする。
問題なければ、次へ進む。

### 4. `sitemap.xml`をダウンロードして、正しいXMLフォーマットか確認する

リポジトリの `sitemap.xml` を開き、
右上の「`Download raw file`」ボタンでダウンロードします。

ダウンロードしたファイルを以下のようなサイトに貼って、
XMLファイルとして正しいフォーマットになっていることを確認します。

https://jsonformatter.org/xml-validator

XMLファイルを貼り付けて、「Valid XML」と表示されればOKです。

### 5. `sitemap.xml`をサーバー管理者に送付する

先ほどダウンロードしたXMLファイルを、サーバー管理者にメールで送付する。
サーバー管理者が後日、FTPでアップロードしてくれるので、作業完了。

## 新規ページがWikiに追加されたとき

**ここから下は、JavaScriptや、XMLのソースコードが読める人向けの手順です。**
ソースコードが読めない人は、この手順を無視して構いません。

スクリプトが自動的に検出し、`sitemap.xml` の末尾に追記します。

```xml
<!-- 未分類（自動追加） -->
<url>
  <loc>https://df.swiki.jp/index.php?新しいページ名</loc>
  <lastmod>2026-04-01</lastmod>
  <changefreq>monthly</changefreq>
  <priority>0.3</priority>
</url>
```

### 必要に応じて手動で調整する（任意）

GitHub上で `sitemap.xml` を直接編集し、以下を調整できます。

- 適切なカテゴリのセクションに移動する（コメントを目印に）
- `priority` を変更する（重要なページは `0.8` など）
- `changefreq` を変更する（頻繁に更新されるページは `weekly` など）

調整後、commitすれば次回以降のスクリプト実行でもその設定が引き継がれます。

## 初期セットアップ（引き継ぎ時に1回だけ行う作業）

### 1. このリポジトリをフォーク or クローンする

GitHubアカウントを用意し、このリポジトリを自分のアカウントに移管またはフォークする。

### 2. Google Apps Scriptプロジェクトを作る

[Google Apps Script](https://script.google.com) で新規プロジェクトを作成し、 `sitemap_updater.js` の内容を貼り付ける。

### 3. GitHubのPersonal Access Token（PAT）を発行する

GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic) → Generate new token

- スコープ: `public_repo` にチェック
- Expiration: 任意（無期限にする場合は `No expiration`）
-- 無期限はセキュリティの観点からオススメしません。そうする場合は、自己責任でお願いします。
-- もし、後任のWiki管理人様がつよつよプログラマーの方でしたら、Fine-grained tokensを使うなど、いい感じにセキュリティ高めてください。

### 4. スクリプトプロパティを設定する

GASエディタ → 左メニュー「プロジェクトの設定」→「スクリプトプロパティ」→「プロパティを追加」

以下の4つを登録する：

| キー | 値 |
|---|---|
| `GITHUB_TOKEN` | 手順3で発行したトークン 例: `ghp_XXXXX`  |
| `GITHUB_OWNER` | あなたのGitHubユーザー名 例: `qwe0001` |
| `GITHUB_REPO` | このリポジトリ名 例: `df-wiki-sitemap` |
| `GITHUB_BRANCH` | マスターブランチ名 例: `main` |

### 5. 動作確認

`updateSitemap` を実行してエラーが出なければセットアップ完了。

## カテゴリの仕組み

`sitemap.xml` 内のXMLコメントがカテゴリの区切りになっています。

```xml
<!-- 乗り物系(航空機系) -->
<url>
  <loc>https://df.swiki.jp/index.php?突撃ヘリ</loc>
  ...
</url>
<url>
  <loc>https://df.swiki.jp/index.php?F-45A戦闘機</loc>
  ...
</url>

<!-- 乗り物系(地上車両系) -->
<url>
  ...
```

スクリプトはこのコメントを保持したままマージするので、カテゴリ構造は自動実行によって壊れません。

---

## 除外されるページ

以下のページはsitemap.xmlに含まれません。

- `Comments/` で始まるコメントページ
- スクリプト内のブラックリストに登録されたページ（テンプレート置き場、Sandbox など）

ブラックリストに追加したいページがある場合は、`sitemap_updater.js` の `CONFIG.BLACKLIST` に追記してください。

## トラブルシューティング

### 「スクリプトプロパティに以下のキーが設定されていません」と表示される

スクリプトプロパティの設定が不足しています。
「初期セットアップの手順4 - スクリプトプロパティを設定する」を確認してください。

### 「GitHub APIエラー HTTP 401」と表示される

GitHubトークンが無効になっています。
トークンを再発行して `GITHUB_TOKEN` を更新してください。
トークンに有効期限を設定した場合、期限切れでも同じエラーになります。

### 「バリデーションエラー：抽出URL数が既存URL数の50%を下回っています」と表示される

Swikiのフェッチに何らかの問題が起きた可能性があります。
Pushは中止されています。
時間をおいて再実行してください。
改善しない場合はSwikiのページ一覧（ https://df.swiki.jp/?cmd=list ）にブラウザで直接アクセスして、正常に表示されるか確認してください。

### 新規追加されたURLがコメント付きで大量に出てきた

正常な動作です。
Wikiに新規ページが追加されると `<!-- 未分類（自動追加） -->` セクションに自動で追記されます。
必要に応じて、あなたが、GitHub上で手動整理してください。
